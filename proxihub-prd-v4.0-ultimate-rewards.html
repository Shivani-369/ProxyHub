🚀 ProxiHub v4.0

Ultimate Hyperlocal Platform • VoiceFirst • Community • Gold Rush • ProxiRewards

📄 Version: 4.0 (Final Production-Ready) 📅 Date: April 2026 🏢 Project Owner: UVFarms, Chennai 🎯 Focus: Inclusive, Gamified, Monetized 5km Discovery

📋 Executive Summary

Core Mission: ProxiHub eliminates the digital visibility gap for India's 60M+ unorganized local vendors by creating a real-time, proximity-first discovery platform within a strict 5km radius—now evolved into an inclusive, community-powered, monetized ecosystem with voice navigation, group buying, neighborhood engagement, gamified urgency, AND rewarded advertising.

Why Now? India's hyperlocal commerce is fragmented: delivery apps ignore unorganized vendors, directories lack real-time data, low-literacy users are excluded, and vendors lack sustainable monetization. ProxiHub bridges this with:

  • 🗣️ VoiceFirst UI: Vernacular voice navigation for Tamil, Hindi, Telugu speakers with low digital literacy
  • 🤝 Community Collective: Group buying & neighborhood order pooling to drive volume for small vendors
  • 🌐 Neighborhood Hub: Hyperlocal community feed for announcements, events, and trust-building
  • 🪙 Local Gold Rush: Gamified time-sensitive deals that create urgency and foot traffic
  • 💰 ProxiRewards: Users earn ₹3-₹10 per ad view; advertisers get guaranteed hyperlocal engaged viewership

Key Differentiator: No other platform combines voice accessibility + community commerce + gamified discovery + rewarded ads + real-time vendor tracking for India's unorganized local economy.

💰 ProxiRewards: Deep Dive

Why ProxiRewards? Creates a sustainable monetization loop: Advertisers pay for guaranteed engaged hyperlocal viewership → Users earn passive income → ProxiHub takes 20-40% commission → More advertisers join → More users earn → Network effects accelerate growth.

🎯 Core Value Proposition

For Users: Earn Passive Income

  • Minimum withdrawal: ₹100 via UPI/bank transfer (24-48h processing)
  • Daily earning limit: ₹100 (prevents abuse, enables meaningful income)
  • Only see ads relevant to your 5km neighborhood
  • Optional: Higher rewards for video ads vs. image ads

For Advertisers: Guaranteed Engaged Viewership

  • Precise 5km hyperlocal targeting: Reach only nearby potential customers
  • Multi-format support: Static images + YouTube-hosted videos (15s-5min)
  • Real-time analytics: Views, completion rate, geographic heatmap, ROI tracking
  • Fraud-protected: Multi-layer detection ensures genuine engaged views
  • Flexible budgeting: Total budget, daily cap, CPV bidding (₹5-₹15)

For ProxiHub: Sustainable Revenue Stream

  • Platform commission: 20-40% of ad spend (₹1-₹4 per view)
  • Example: Advertiser pays ₹10 → User earns ₹6 → ProxiHub keeps ₹4
  • Scalable model: More users = more ad inventory = more advertiser demand
  • Defensible moat: Hyperlocal + fraud prevention + engaged user base
  • Projected Year 1 Revenue: ₹9.45L - ₹60.75L (conservative to optimistic)

🔄 User Earning Flow

Step-by-Step Journey:
  1. Discovery: User opens "Earn Rewards" tab → System fetches nearby ads (5km radius) → Display: Ad preview + Reward amount + Duration
  2. View Initiation: User taps ad card → Full-screen viewer opens → Countdown timer starts (e.g., 30 sec) → Progress bar shows completion %
  3. Active Viewing: User must keep app in foreground → Background detection pauses timer → GPS validation continues (stay in radius) → Optional: Interaction prompts (prevent bots)
  4. Completion & Reward: Timer reaches required duration → "Claim Reward" button activates → User taps to confirm → Reward credited to wallet → Push notification: "₹5 earned!"
  5. Withdrawal: Balance ≥ ₹100 → User requests withdrawal → KYC verification (if first time) → Processing (24-48 hours) → Funds transferred via Razorpay Payouts

🔄 Advertiser Campaign Flow

Step-by-Step Journey:
  1. Onboarding: Sign up as Advertiser → KYC verification (PAN/GST) → Wallet top-up via Razorpay (min ₹1000)
  2. Campaign Creation: Select ad type (Image/YouTube Video) → Upload creative or paste YouTube URL → Set target location (pin drop + radius) → Configure budget & bidding (Total budget, CPV, Daily cap) → Set campaign dates
  3. Approval: Submit for admin review → Content moderation check → Budget verification → Approval within 24 hours
  4. Live Campaign: Ad enters discovery pool → Shown to users in target area → Real-time budget deduction → Performance tracking active
  5. Optimization: View analytics dashboard → Adjust targeting/budget → Pause or extend campaign → Download performance reports

🛡️ Fraud Detection: 5-Layer Architecture

Layer 1: Device Fingerprinting

  • FingerprintJS for unique device identification
  • Root/jailbreak detection (+40 risk score)
  • Emulator detection (+60 risk score)
  • Multi-account device check (+30 risk score)
  • VPN/Proxy detection via IP quality APIs (+25 risk score)
  • Threshold: Risk score ≥70 = auto-reject

Layer 2: Geolocation Validation

  • Haversine formula: Verify user GPS within ad radius
  • IP vs GPS mismatch detection (+40 risk score if >50km)
  • Mock location app detection (Android) (+70 risk score)
  • Impossible travel speed check (+35 risk score if >100km/h)
  • Real-time GPS validation during view session

Layer 3: Behavioral Analysis

  • Completion rate analysis: Bots often have ~100% completion (+30 risk)
  • Exact timing pattern detection: Too many exact-duration views (+40 risk)
  • Interaction score: Less than 1 interaction/10s = passive viewing (+25 risk)
  • Background time check: App in background >30% of view time (+35 risk)
  • Viewing velocity: >10 views/hour = suspicious (+45 risk)

Layer 4: Rate Limiting (Redis)

  • Daily earning limit: ₹100 max per user per day
  • Hourly view limit: Max 10 views per user per hour
  • Device limit: Max 20 views per device per day
  • IP limit: Max 50 views per IP per hour
  • Atomic Redis operations with pipeline for performance

Layer 5: Pattern ML (Phase 2)

  • Historical pattern analysis for anomaly detection
  • Clustering to identify bot farm behavior
  • Real-time scoring with adaptive thresholds
  • Feedback loop: Flagged views improve model accuracy
  • Admin dashboard for manual review of edge cases

  • Redis 7+: Geo-queries, caching, rate limiting, BullMQ for jobs, Pub/Sub for real-time
  • Socket.io + Redis for real-time chat, rush updates, hub notifications, ad view tracking

Infrastructure

  • Offline-First: Service workers + background sync + conflict resolution
  • Observability: Prometheus + Grafana + Sentry for monitoring
  • ProxiRewards: Redis pipelines for high-throughput rate limiting; WebSocket clustering for view tracking

Key Integrations

⚠️ Risks & Mitigation (v4.0)

Risk Mitigation Strategy
ProxiRewards Fraud
Bot farms or fake views draining advertiser budgets		 • 5-layer fraud detection pipeline (Device + Location + Behavior + Rate Limits + ML)
• Real-time WebSocket heartbeat validation
• Manual review queue for edge cases
• Insurance fund: 5% of commission revenue reserved for fraud losses
• Transparent reporting: Advertisers see fraud-filtered metrics
Regulatory Compliance
Rewards classified as "income" requiring TDS/GST		 • Legal review: Position rewards as "engagement incentives", not salary
• KYC for withdrawals >₹10,000 (PAN verification)
• Auto-TDS deduction (1%) on winnings >₹10,000/year per user
• Clear terms: "Rewards are promotional incentives, not taxable income under ₹10,000"
• DPDP Act 2023 compliance: Data minimization, consent logs, right to deletion
Cash Flow Management
Users withdraw before advertisers pay or campaigns complete		 • Prepaid model: Advertisers must top-up wallet before campaigns go live
• Minimum withdrawal threshold: ₹100 (reduces transaction frequency)
• Processing time: 24-48 hours for withdrawals (buffer for reconciliation)
• Reserve fund: Maintain 20% of commission revenue as operational buffer
• Auto-pause campaigns when advertiser balance < ₹500
Advertiser Adoption
Insufficient ad inventory limits user earning potential		 • Free trial credits: ₹500 for first campaign to lower barrier
• On-ground sales team: Target local businesses in pilot cities
• Case studies: Share pilot results (engagement rates, foot traffic lift)
• Partnership program: Local business associations, Chambers of Commerce
• Lower CPV for early adopters: ₹5-₹7 vs. standard ₹7-₹15
User Churn
Initial excitement fades; users stop engaging with rewards		 • Gamification: Badges for streaks ("7-Day Earner"), milestones ("₹500 Total")
• Daily challenges: "Watch 3 ads today → Bonus ₹5"
• Referral program: "Invite friend → Both get ₹20 bonus"
• Personalization: Show ads matching user interests (groceries, repairs, food)
• Community integration: "Your neighborhood earned ₹X this week" in Hub feed
YouTube API Limits
Rate limiting or quota exceeded for video validation		 • Cache video metadata (title, duration, thumbnail) for 24 hours
• Request increased quota from Google for production use
• Fallback: Manual validation workflow for high-volume advertisers
• Monitor quota usage with alerts at 50%/80%/95% thresholds
• Alternative: Support direct video upload (Cloudinary) for non-YouTube ads (Phase 3)
